Glassmorphism Navigation Bar

Premium glassmorphism effect with automatic light/dark theme support. The navigation bar features backdrop blur, smooth transitions, and auto-hide on scroll.

Glassmorphism Effect

This demonstrates the reusable .glass utility class. The navigation bar at the top of this page uses the same glassmorphism effect.

Features:

  • Automatic light/dark theme adaptation
  • Backdrop blur with saturation enhancement
  • Browser fallback for unsupported browsers
  • Smooth scroll-based auto-hide behavior
Navigation Bar (Automatic):
<header class="site-header">
  <nav class="nav">
    <!-- Navigation content -->
  </nav>
</header>

Reusable Glass Component:
<div class="glass">
  <p>Content with glassmorphism effect</p>
</div>

Theme Toggle: Use the theme toggle in the navigation bar above to see the glassmorphism effect in both light and dark themes.

Scroll Behavior: Scroll down to see the navigation bar auto-hide, then scroll up or pause to see it reappear.

CSS Variables: Customize the effect by modifying --glass-bg-light, --glass-bg-dark, --glass-blur, and --glass-saturation in your stylesheet.

Accessibility & Touch Optimization

Accessibility Features

  • Focus States: All interactive elements have visible focus indicators using focus-visible with 3px outline and 2px offset.
  • Keyboard Navigation: Full keyboard support for all components. Modals trap focus and can be closed with Esc.
  • ARIA Support: Modals use aria-modal and role="dialog". Accordions use aria-expanded. Tabs use proper ARIA roles: tablist, tab, tabpanel.
  • Color Contrast: All text meets WCAG AA standards (4.5:1 for normal text, 3:1 for large text).
  • Semantic HTML: Proper heading hierarchy, landmark regions, and semantic elements throughout.
  • Screen Reader Support: Tooltips visible on focus, proper labels for form controls, descriptive alt text patterns.

Touch Device Optimization

  • Touch Targets: All interactive elements meet WCAG 2.5.5 minimum size of 44×44px (comfortable 48px).
  • Spacing: Increased spacing between touch targets (minimum 8px) to prevent accidental taps.
  • Tap Highlight: Removed default iOS tap highlight for cleaner interactions.
  • Touch Scrolling: Smooth scrolling with -webkit-overflow-scrolling: touch for better mobile performance.
  • Form Controls: Checkboxes and radio buttons enlarged to 20×20px on touch devices.
  • Active States: Visual feedback on touch with scale transforms and background changes.

Interactive States

All interactive components include comprehensive state management:

Button States

  • :hover - Subtle lift with shadow
  • :active - Pressed state (scale down)
  • :focus-visible - Keyboard focus ring
  • :disabled - Reduced opacity, no pointer events

Form States

  • :focus - Border color change + focus ring
  • :disabled - Grayed out, not interactive
  • :invalid - Error styling (when implemented)
Example: Button with all states
/* Example: Button with all states */
.btn {
  /* Base styles */
  min-height: 44px; /* Touch target */
  transition: transform 0.2s ease;
}

.btn:hover { transform: translateY(-1px); }
.btn:active { transform: translateY(0); }
.btn:focus-visible { outline: 3px solid var(--focus-ring-color); }
.btn:disabled { opacity: 0.5; cursor: not-allowed; }
